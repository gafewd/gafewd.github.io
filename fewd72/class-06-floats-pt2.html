Floats/Clears Review

Floats - Review

• Enable complex layouts by altering a block element's "flow" in the document

• Floated elements will shift left or right, until:

⇒ They reach the side of their parent

⇒ They encounter another floating element

• Text and Inline elements will now wrap around it

• Example

Clears - Review

• How we "fix" or control floating elements

• Various approaches to clearing elements:

⇒ clear CSS property

⇒ overflow: hidden

⇒ clearfix

Clears - The "Clear" CSS Proprety

• clear is a CSS property that takes:

⇒ left

⇒ right

⇒ both

• Tells non-floating elements where floats are not allowed (left, right, both sides)

Clears - overflow: hidden

• The overflow CSS property controls what happens to content that breaks outside of it's container (ex.)

• Takes the following values: visible, hidden, scroll, auto

• If hidden or auto is applied to a element containing floated elements, will cause the element to extend to accompany any floating children (ex.)

• More info

Clears - clearfix

• Applying overflow may cause undesirable behavior

• Clearfix to the rescue:


.clearfix:after {
  content: "";
  display: table;
  clear: both;
}

• Works on collapsed elements only!
