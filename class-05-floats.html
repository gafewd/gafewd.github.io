Floats

Floats - Origins

• CSS positioning property

• Enables advanced/customized layout

• Derived from text wrapping in print:

Floats - Origins

• Similar to print, can instruct HTML elements to float:


	img {
		float: left;
	}

• Valid values: left, right, none, inherit

• CodePen exercise

Floats - Some Rules....

1 • A left-floated box will shift to the left until its leftmost margin edge touches either the edge of the containing block, or the edge of another floated box ⇒ Example

2 • A floated element should always have an explicit width set. If no width is set, it will collapse to be the width of the content within it (which may vary browser-to- browser / depend on font-size) ⇒ Example

3 • Block level elements above a floated element will not be affected by it. However, elements below will wrap around the floated element. To fix this, you need to do what we call "clearing" the float ⇒ Example

Clearing Floats

Clearing Floats

• Floated objects do not add height to their containing objects; this creates two problemos:

⇒ Objects further down the page will render incorrectly

⇒ If there are only floated objects within an object, it will collapse

⇒ Example

Clearing Floats - The Overflow Method

• Add the following to collapsed element:

overflow: hidden;

• Tells the browser to pay attention to the height of the content

Clearing Floats - The Clear Class & Object

• Define the following in your CSS:


	.clear {
		clear: both;
	}

• Create the following element in your HTML; place it AFTER the floated elements:

Clearing Floats -The Clearfix Class

• Define the following in your CSS:


	.clearfix:after {
	    content:"";
	    display:table;
	    clear:both;
	}

• Add the class to the element that has collapsed
