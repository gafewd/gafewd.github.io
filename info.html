FEWD Overview

FEWD Overview

• Frontend Web Development:

⇒ The planning and building of websites and web applications to be rendered in the browser

• Toolbox:

⇒ HTML:

• HyperText Markup Language

• Describes layout / structure (eg position of a nav bar)

⇒ CSS:

• Cascading Style Sheets

• Describes the presentation of the layout (eg color of a nav bar)

⇒ JS:

• JavaScript

• Describes the behavior of the layout (eg what happens when you click on the nav bar)

FEWD Overview: FEWD vs BEWD

• Frontend Web Development:

⇒ Developers for the client (aka the browser, aka the frontend)

⇒ Build what users interact with (what they see)

⇒ Works with HTML, CSS, and a lot of JavaScript

• Backend Web Development:

⇒ Develops on the server (aka the backend)

⇒ Build the services behind the scenes

⇒ Common languages: Java, Python, Ruby

⇒ Secretly jealous of frontend developers

FEWD Overview: FEWD vs BEWD
FEWD Overview: Typical Day

Frontend developers interact with nearly all levels of the organization:

• Meet with design team to assess feasibility of a new, more modern nav bar

• Analyze user data with customer service/sales to determine why a feature is not being adopted

• Discuss feature roadmap with product managers

• Direct blame at backend developers when a feature breaks

Course Overview

⇒ HTML

⇒ CSS

⇒ JavaScript

⇒ jQuery (a very popular JavaScript library)

• Ten-week course:

⇒ Six hours / week of in-class instruction

Who are my classmates?

Students

Students generally come from one of five backgrounds:

1) Entrepreneurs seeking to create an online prototype

⇒ Translate business ideas into a prototype

⇒ Communicate with a technical co-founder / employees

⇒ Pick up tech-slack where they can

2) Career changers

⇒ Curious about breaking into development

⇒ Testing the waters before taking an immsersive course

⇒ Note: FEWD is not intended to land you a full-time job

Students

3) Backend Web Devs

⇒ Backend devs looking to learn about client-side development

⇒ Want to see what all the noise is about

4) Designers + Project Managers

⇒ Want to better communicate with frontend developers

⇒ Understand what design is possible in the browser

⇒ Better estimates for project completion

5) Curious Hobbyists

⇒ Have hacked with HTML/CSS/JS for a while

⇒ Looking to cement their knowledge and fill in gaps
